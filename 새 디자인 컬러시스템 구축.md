# 새 디자인 컬러시스템 구축

## 프로젝트 소개

- 단순한 **컬러 팔레트 생성기**가 아니라
- 디자인 토큰을 생성하는 **규칙 시스템 (Formula-driven Design System Engine)** 을 만드는 것

## 핵심 컨셉

- 컬러를 저장하는 방식이 아니라 **컬러를 생성하는 수식을 저장**하는 디자인 시스템
- 전통적인 RGB 방식이 아니라 인간 시각 기반 색공간의 **OKLCH** 를 적용

## 접근 방법

**UI 툴 개발 방식 + 디자인 시스템 방법론 + 컬러 사이언스**


# 1. 프로젝트의 핵심 개념 정의

### 기존 디자인 시스템 방식

```
Brand color → palette → token → UI
```

디자이너가 수십~수백개의 색을 직접 정의.

### 이 프로젝트의 방식

```
Brand color
     ↓
Color Engine (OKLCH + Formula)
     ↓
Derived Tokens
     ↓
UI
```

핵심 특징

1. 색을 생성하는 규칙을 저장한다
3. 모든 토큰은 키컬러에서 파생된다
4. 모든 색은 OKLCH 기반 계산

# 2. 전체 시스템 구조

전체 구조는 5개 레이어로 구성

```
1 Brand Input Layer
2 Color Engine (OKLCH)
3 Token Generator
4 Formula Editor
5 Design System Export
```

# 3. Color Engine (OKLCH)

이 프로젝트의 **가장 중요한 부분,** OKLCH 기반 색 계산

### 기존 RGB / HSL 문제

- 밝기 균일성 없음
- 컬러 스케일이 불안정

### OKLCH 장점

- 인간 시각 기반 색공간
- 밝기 조절이 자연스러움
- UI 컬러 스케일 생성에 적합

### 기본 구조

```
Color {
  L : Lightness
  C : Chroma
  H : Hue
}
```

### 예시 브랜드 컬러

```
Primary : oklch(0.62 0.21 250)
Secondary : oklch(0.68 0.18 320)
Accent : oklch(0.70 0.17 150)
```

# 4. Token Formula

각 토큰은 색이 아니라 **수식으로 정의된다**

### 예시 Token Formula  예시

토큰을 구성하는 네이밍 공식도 설정화면에서 설계 및 수정이 가능한 UI 제공

### primary-button-normal

```
primary
```

### primary-button-hover

```
L = primary.L - 0.05
C = primary.C
H = primary.H
```

### primary-button-soft

```
L = primary.L + 0.20
C = primary.C * 0.3
H = primary.H
```

### primary-button-border

```
L = primary.L + 0.10
C = primary.C * 0.5
```

### 예시

```
color.primary.hover

```

### 규칙

```
category.role.state
```

```

```

# 8. Formula Editor (핵심 UX)

더 자연스러운 색상생성을 위해서 필요시 디자이너가 **수식을 수정할 수 있어야**

### 예시

```
Token: button-hover

Formula:

L = primary.L - 0.08
C = primary.C * 0.9
H = primary.H
```

수정하면 → 모든 UI 업데이트

### 라이브 업데이트 구조

```
Formula change
   ↓
Color engine
   ↓
Token regeneration
   ↓
UI update
```

# 9. Formula DSL (Domain Language)

수식을 사람이 읽기 쉽게 만들 필요. 


### 예시 DSL

```
lighten(primary, 10%)
desaturate(primary, 20%)
rotateHue(primary, 15)
mix(primary, neutral, 30%)
```

또는

```
primary.l + 0.1
primary.c * 0.5
primary.h + 15
```

# 10. Neutral Color Engine

Neutral 팔레스 자동생성은 UI에서 가장 변별력을 만들기 어려운 부분이라는 생각

(거의 키컬러와 관계 없는 흑백계열 색으로 구성되는 경우가 많다보니….)

### 방법

브랜드 컬러 hue를 활용

```
neutral.h = primary.h
neutral.c = 0.02
```

또는 완전 중립

```
neutral.c = 0
```

# 11. 접근성 계산

생성된 컬러에서 매우 부자연스럽다고 판단되는 컬러나 매칭이 생길경우 알림이나 보완경고.

### 자동 contrast 계산

```
WCAG contrast
```

예

```
text-primary vs background
```

자동 경고

```
⚠ contrast fail
```

# 12. 디자인 시스템 저장 방식

가장 중요한 구조

## 저장 데이터의 핵심은 초기값(키컬러들)와 수식

물론 수식의 결과값으로 UI에 최종적으로 반영되는 컬러값도 저장은 되겠지만..

### 예시 JSON

```
{
 "brandColors": {
   "primary": "oklch(0.62 0.21 250)"
 },

 "tokens": {
   "button-hover": {
     "formula": "primary.l - 0.08"
   }
 }
}
```
